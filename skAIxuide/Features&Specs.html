Environment & Persistence

Zero-Server Persistence Layer

SkAIxu eschews traditional cloud file systems for a Local-First approach. All active file buffers, folder structures, and chat history are serialized into JSON and stored within the browser's IndexedDB (via `idb-keyval`). This ensures:

  • Data Sovereignty

    Your code never leaves your device unless you explicitly prompt the AI.

  • Instant State Recovery

    Reloading the browser restores the exact pixel-perfect state of your editor, selection, and history instantly.

  • Local Versioning Stack

    An in-memory stack tracks every keystroke and AI generation, enabling infinite Undo/Redo traversal during a session.

Visual Augmentation & Injection

The IDE employs a sophisticated postMessage bridge to communicate between the editor logic and the sandboxed preview `iframe`. This enables the "Draw Selection" tool:

Workflow Logic

  1. 1 User toggles "Draw Mode" -> Script injected into iframe.
  2. 2 Mouse events capture coordinates (X,Y) and render overlay box.
  3. 3 `document.elementFromPoint(x,y)` identifies the DOM node.
  4. 4 Node's `outerHTML` is serialized and sent to parent.
  5. 5 AI context is updated to FOCUS exclusively on that element.
