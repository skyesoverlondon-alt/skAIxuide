4) Critical findings (security + correctness)

  • FIXED (P0): PUT /projects/:id previously performed file upserts/deletes before verifying the project belonged to the caller’s org. That enabled cross-tenant file tampering if an attacker guessed another project UUID. Hotfix now gates writes behind a target CTE filtered by org_id.
  • P1: Users can belong to multiple orgs but login picks only one membership (highest role). Enterprises expect org switching and explicit org selection.
  • P1: No password reset, no email verification, no lockout/backoff, no MFA.
  • P2: Rate limit is best-effort (in-memory per lambda instance). For abuse control, use Redis/Upstash + token bucket.
  • P2: No lockfile/SBOM/tests/CI → supply-chain and change-control gaps for regulated buyers.
