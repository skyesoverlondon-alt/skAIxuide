Self-host Infrastructure Your accounts, your control

How you run it on your own

“Self-hosted” means your organization owns the domain, hosting accounts, secret store, and (if used) the database. The platform is delivered aligned to your chosen deployment model.

Option A

Netlify-first

Static assets + serverless functions + managed DB. Fast deploys, clean secret management.

Option B

Cloudflare-first

Edge runtime for governance/policy + static hosting. Strong request control posture.

Option C

Vercel / Serverless

Serverless APIs + static assets + your DB. Works well in modern enterprise stacks.

Option D

Fully self-managed

VM/containers + reverse proxy + Postgres. Highest autonomy, requires ops ownership.

Core components
1) Platform UI • 2) Governance layer • 3) Storage (local + optional server features) • 4) Secrets/config • 5) Domain/TLS
