WHAT IS
NEURAL SPACE PRO

Neural Space Pro is a production-grade, multi-modal AI research and analysis environment. It is not a wrapper around a chatbox. It is a fully engineered browser-native workspace capable of ingesting PDFs, spreadsheets, and images alongside natural language prompts, rendering mathematical equations, code with syntax highlighting, technical diagrams, and streamed AI responses in real time — all governed through the Kaixu Gateway.

Every rendering engine, every file parser, every streaming SSE client, every diagnostic channel — built from scratch into a single cohesive professional workspace. The intelligence layer runs through the Kaixu Gateway, which means operator governance, key isolation, and a single authenticated route to all AI capabilities. Neural Space Pro is the research and analysis workstation of the SkAIxu ecosystem: the environment where operators do serious work against serious data.

This tool is accessed from inside SkAIxu IDE's app switcher. It is one of the premium decision-making surfaces of the SOLEnterprises intelligence stack. Its separate valuation reflects the distinct engineering investment it represents.

Section 02

Core Chat Architecture & SSE Streaming Engine

The most technically complex component of Neural Space Pro is its streaming SSE client. This is not a simple EventSource — it is a fetch + ReadableStream implementation required because EventSource does not support custom authorization headers over POST. The entire streaming pipeline is custom-engineered.

  • Dual-base fallback: the streaming client attempts /api (Netlify proxy / local server.py) first, falls back to the direct Kaixu Gateway at kaixugateway13.netlify.app — no single point of failure.
  • Idle watchdog: a setTimeout-based watchdog fires after 20 seconds of stream inactivity and cancels the reader, surfacing a clear diagnostic error rather than hanging silently.
  • First-data watchdog: a separate 15-second timeout fires if no data whatsoever arrives from the gateway — protecting operators from invisible stream failures.
  • Abort controller: operators can interrupt a running stream mid-generation via the Interrupt button, immediately cancelling the fetch without leaving dangling connections.
  • Event parsing: the buffer is split on \n\n, lines parsed for event: and data: fields — handling meta, delta, done, and error event types from the gateway.
  • Token-level diagnostics: on stream completion, input/output token counts and elapsed time are broadcast through the kaixu_events diagnostics channel.

This is the kind of streaming implementation that takes multiple engineering iterations to get right. Idle timeouts, dual-base fallbacks, proper SSE buffer handling, and abort semantics are each non-trivial. Together they represent a mature, battle-tested implementation.

Section 03

Multi-Modal Attachment System

Neural Space Pro accepts three categories of file input — images, PDFs, and spreadsheets — and processes each into a format the AI can operate on. This is not a gallery of file icons; it is a genuine document intelligence pipeline.

  • PDF parsing: PDF.js is loaded with a dedicated worker (pdf.worker.min.js). Each page is iterated via getPage, text content extracted via getTextContent, and the full text assembled and injected into the AI prompt as a named context block — [FILE: filename.pdf].
  • Spreadsheet parsing: SheetJS reads the binary ArrayBuffer, opens the workbook, converts the first sheet to CSV via sheet_to_csv, and injects it into prompt context. Excel, XLS, and CSV are all supported.
  • Image handling: images are read as DataURL via FileReader, the base64 payload extracted, and passed as an image attachment type to the AI payload — ready for multimodal providers.
  • Attachment strip UI: chips render with thumbnails (images) or file-type icons (PDFs, spreadsheets), with per-file removal controls and automatic strip collapse when empty.
  • All three parsers are async, properly awaited, and wired into the same attachment state array that feeds the AI call — context-injection is automatic once files are attached.
Section 04

Rendering Suite — Math, Diagrams, Code, Markdown

Neural Space Pro renders four distinct output types natively inside AI responses. Most tools render markdown. This one renders mathematics, technical diagrams, syntax-highlighted code, and markdown simultaneously — all passing through a single marked.parse pipeline.

  • KaTeX (mathematics): marked-katex-extension integrated into the marked pipeline renders both inline $...$ and block $$...$$ LaTeX expressions — research-grade output for quantitative work.
  • Mermaid (diagrams): flowcharts, sequence diagrams, and entity-relationship diagrams render live in AI responses — system architecture documentation and data modelling inside the workspace.
  • Prism (code): Python and JavaScript syntax highlighting via Prism with the Tomorrow Night theme — code blocks in AI responses are immediately readable production output, not monochrome walls of text.
  • Marked (markdown): the core rendering layer processes headings, lists, tables, blockquotes, and inline formatting — AI responses arrive formatted, not raw text requiring manual parsing.
  • Prism.highlightAllUnder(div) is called after both streaming completion and session load — code blocks highlight correctly whether the response was streamed or loaded from history.

The integration of all four rendering engines into a single coherent DOM output layer is not trivial engineering. Each library handles its own DOM manipulation. Making them coexist inside streamed markdown output — including re-rendering correctly after stream completion — is the kind of detail that distinguishes an engineered product from an assembled one.

Section 05

Live Code Canvas & Preview Panel

Neural Space Pro ships with a split-panel canvas that allows operators to take AI-generated code directly into an editor and execute it in a live iframe preview — without leaving the workspace or opening a second tool.

  • Canvas panel: slides in from the right at 50% width via CSS transition (translateX + width + opacity), toggled by toggleWorkspace(true/false). Panel state is encoded on #app-body via canvas-open / canvas-closed classes.
  • Editor tab: a raw textarea allows operators to edit AI-generated code before running it.
  • Live Run tab: the canvas-editor content is set as iframe.srcdoc — the code executes in a sandboxed iframe with full browser rendering. HTML, CSS, and JavaScript all execute.
  • Copy from response: a toCanvas(btn) function extracts the .prose innerText from any AI response and loads it into the canvas editor — one action, no clipboard required.

The canvas panel is a complete mini-IDE embedded in the research workspace. This is the feature that distinguishes Neural Space Pro from a chat interface — it is a work environment, not a conversation thread.

Section 06

Kaixu Gateway Integration & Governance

Neural Space Pro is fully governed by the Kaixu Gateway — the same single-key, single-authorisation-surface architecture that governs all tools in the SkAIxu ecosystem. The governance layer is wired at the function level, not the configuration level.

  • Key auto-injection: injectKeyFromServer() fetches from /api/kaixu-key on load — in production, Netlify Functions return the key from environment variables, keeping secrets server-side and never in the client bundle.
  • Fallback chain: if server injection fails, the key is read from localStorage.getItem('KAIXU_VIRTUAL_KEY'). If no key exists in local or dev environments, the user is prompted interactively. The system degrades gracefully at every step.
  • KAIXU_IS_LOCAL detection: the boolean is computed from hostname at runtime — local server.py proxy is used in dev, Netlify Functions in production, no conditional code in the business logic.
  • Provider-agnostic payload: the AI request is structured as { provider: 'gemini', model: 'gemini-2.0-flash', messages: [...] } — the gateway selects and authenticates the provider, Neural Space Pro never touches a provider API key directly.
  • KAIXU_IDENTITY injection: a system-level identity prompt is always prepended — operator-configured system prompts layer on top, never replacing the identity lock.
Section 07

Diagnostic & Observability System

Neural Space Pro runs a production-grade observability layer — BroadcastChannel, structured event types, 20-second heartbeats, and a client-side error surface. This is the level of operational visibility expected in enterprise tooling, built natively into a browser app.

  • BroadcastChannel diagnostics: all significant events — key updates, stream start/complete/error, token counts, heartbeat readings — are emitted on the kaixu_events channel. Any open tab in the ecosystem can listen and surface these in real time.
  • Heartbeat loop: every 20 seconds, the system pings the server key endpoint, checks gateway HEAD availability, reads localStorage key state, and checks navigator.onLine — all results broadcast and logged. Operators always know whether the environment is functioning.
  • Client error surface: #client-error-panel renders error details with full environment context (describeClientEnv()) — protocol, host, path, base mode, service worker state, UA — directly in the UI rather than in the browser console.
  • Unhandled rejection handler: window.addEventListener('unhandledrejection') catches any promise rejection not caught elsewhere, routes it through renderClientError, and surfaces the full stack trace.
  • Remote log batching: in production (non-local environments), log events are batched and flushed every 5 seconds to /.netlify/functions/logs via POST — with sendBeacon as the unload safety net.
Section 08

Knowledge Base, Session Persistence & History

Neural Space Pro maintains full session persistence through localStorage — no backend session layer required. Conversation history, pinned knowledge base items, system prompts, and API key state all survive browser reloads.

  • Chat index: kaixu_chats_v1 stores an array of chat metadata (id, title, updatedAt). New sessions are prepended; the index is rendered in the sidebar on every change.
  • Per-session messages: each conversation is stored as kaixu_chat_{id} — an array of message objects with role, content, attachments (name and type), and timestamp. Full fidelity history reload on click.
  • Knowledge Base drawer: a full-width right-side drawer with pinned items persisted separately — operators curate a reference set that carries across sessions.
  • Session reload: loadSession(id) reads the message store, calls addMessageUI for each message, rebuilds the conversation array, and re-runs Prism.highlightAllUnder — code blocks highlight correctly in historical sessions.
  • System prompt persistence: the operator-configured system prompt is stored as kaixu_sys and prepended to every AI call — custom instructions survive across sessions.
Section 09

PWA, Infrastructure & Code Work That Drives Value

Neural Space Pro is a PWA — installable on desktop and mobile via the browser's install prompt. PWA infrastructure, service worker registration, Netlify configuration, and the localStorage contract layer are all engineering hours that drive asset value.

  • PWA manifest: manifest.json defines name, short name, description, theme and background colours, display mode (standalone), start URL, and icon sets at 192px and 512px — full PWA installation spec.
  • Service worker: sw.js is registered at window load — offline-first caching strategy makes Neural Space Pro resilient to connectivity drops during long research sessions.
  • Netlify Functions integration: both the key injection endpoint (/api/kaixu-key) and the log batching endpoint (/.netlify/functions/logs) are part of the deployed infrastructure. The netlify.toml redirect rules make /api resolve correctly in production.
  • localStorage contract: KAIXU_VIRTUAL_KEY is the shared key schema readable by SkAIxu IDE and all peer tools — Neural Space Pro participates in the ecosystem-wide governance architecture as a first-class member.
  • Local development proxy: the KAIXU_IS_LOCAL detection and /api base URL allow the full production API flow to work against a local server.py proxy without any code changes.
